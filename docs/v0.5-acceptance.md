# v0.5 production acceptance

Run this checklist on the Windows production computer with Planning Center,
Playback MIDI, and ProPresenter available. Do not paste credentials into test
logs or screenshots.

## First launch

1. Start the backend and dashboard without any `STAGEPILOT_*` environment variables.
2. Confirm the first-launch checklist reports the unfinished setup steps.
3. Open **StagePilot backend**, review timezone, log level, and server port, then save.
4. Open **Planning Center**, enter the PAT credentials, test the connection, load
   service types, choose the production service type, and save.
5. Open **MIDI / Playback**, confirm note `112`, velocities `100`–`105`, the
   intended channel, and debounce, then save.
6. Open **ProPresenter**, enter the API host, port, countdown timer, and timeout,
   then save.
7. Restart StagePilot.

Expected: no PowerShell configuration is required, the real integrations start,
and no production configuration panel offers a demo or simulated source.

## Persistent configuration

1. Refresh MIDI inputs and select the Playback input.
2. Restart the backend and dashboard again.
3. Confirm the general values, Planning Center application ID and masked-secret
   state, service type, MIDI input and cue settings, and ProPresenter settings
   are populated.
4. Open Windows Credential Manager and confirm a generic credential for the
   `StagePilot` service exists. Do not reveal its value.

Expected: `%APPDATA%\StagePilot\settings.json` contains ordinary configuration
but no Planning Center secret, and all selections survive the restart.

## Live workflow

1. Confirm Planning Center lists the real service types.
2. Load today's service, or the nearest upcoming service when today has none.
3. Confirm songs and durations appear in Planning Center order.
4. Send Playback note `112` with each velocity from `100` through `105` and
   confirm the matching StagePilot and ProPresenter action.
5. Confirm the setup checklist reaches **6 of 6 complete**.

## Last-known-good outage

1. Complete one successful Planning Center refresh.
2. Stop StagePilot.
3. Temporarily block Planning Center network access without deleting settings or
   credentials.
4. Start StagePilot and wait for the refresh attempt to fail.
5. Restore network access immediately after recording the result.

Expected: StagePilot loads the cached plan, labels it stale, displays a warning,
and does not report production readiness until a fresh refresh succeeds.

## Sign-off

Record the date, StagePilot commit, Windows version, Planning Center service
type, MIDI device name, ProPresenter version, and pass/fail result. Never record
the PAT secret.
